# Security Policy

Orifold is a local-first macOS application. It performs all document processing
on-device and makes no network calls to handle user files — nothing about a
user's documents leaves their Mac. We take the security and privacy of that
promise seriously.

## Supported Versions

Orifold is in active beta. Security fixes are applied to the latest released
version.

| Version | Supported          |
| ------- | ------------------ |
| Latest release (0.8.x) | :white_check_mark: |
| Older pre-releases     | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.**

Instead, report them privately using one of the following:

- **GitHub Security Advisories** — use the
  ["Report a vulnerability"](https://github.com/udhawan97/Orifold/security/advisories/new)
  button on the repository's Security tab (preferred).
- **Email** — send details to **umangdhawan97@gmail.com** with the subject line
  `Orifold Security`.

Please include as much of the following as you can:

- A description of the vulnerability and its potential impact
- Steps to reproduce, or a proof-of-concept
- The Orifold version and macOS version affected
- Any relevant crash logs or sample files (please redact sensitive content)

## What to Expect

- **Acknowledgement** within 5 business days.
- **An assessment** and expected timeline for a fix once triaged.
- **Coordinated disclosure** — we'll work with you on timing and will credit you
  in the release notes if you wish.

Thank you for helping keep Orifold and its users safe.
